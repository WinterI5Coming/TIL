# ML-Theory-to-Practice

머신러닝 이론을 단순히 사용하는 것이 아니라,  
**수식 → 직접 구현 → 실험 → 해석**의 과정을 통해 이해하는 프로젝트입니다.

이 프로젝트의 핵심 목표는 모델 성능을 비교하는 것이 아니라,  
**모델이 왜 그렇게 동작하는지 설명할 수 있는 수준의 이해를 얻는 것**입니다.

---

## 1. 프로젝트 목표

이 프로젝트에서는 다음을 “실험으로 검증”합니다.

- 생성모델 vs 판별모델의 차이
- 데이터 분포가 모델 성능에 미치는 영향
- 학습 방식(최적화 방법)의 차이
- 모델의 일반화 능력과 과적합
- 데이터 전처리와 특성 공학의 영향

---

## 2. 핵심 학습 방식

가정 → 데이터 생성/선택 → 직접 구현 → 실험 → 결과 → 해석

---

## 3. 현재 진행 단계

### GDA vs Logistic Regression (From Scratch)

- Synthetic Gaussian 데이터 생성
- GDA 직접 구현
- Logistic Regression 직접 구현
- decision boundary 비교
- 결과 해석

---

## 4. 핵심 원칙

- 반드시 직접 구현 후 라이브러리 비교
- 결과보다 이유를 설명하는 것이 목표
- 항상 데이터와 모델을 함께 해석

---

## 5. 최종 목표

- 모델의 가정 설명 가능
- 결과를 이론과 연결 가능
- decision boundary 이해
- 일반화와 과적합 설명 가능
