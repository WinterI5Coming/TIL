# 함수형 그래프(Functional Graph)

---

## 1) 개념 정의
- 함수형 그래프란?
  - 각 노드에서 **나가는 간선이 정확히 1개인 그래프**를 의미한다.

- 해당 그래프의 특징은 다음과 같다:
  - 전체 그래프는 여러 개의 **사이클 + 트리 구조**로 이뤄진다.
  - 모든 노드는 결국 **어떤 사이클로 흘러 들어간다.**

---

## 2) 구조
어떤 노드에서 출발해서 계속 따라가 보면,\
각 노드는 다음 노드가 무조건 하나가 있기 때문에, 길이 끊기지 않는다.
<br>

그런데 노드 개수는 유한하기 때문에,\
계속 가다 보면 언젠가는 **이미 한 번 방문한 노드를 다시 만나게 되어있다.**
<br>

그 순간부터는 원을 도는데, 그게 바로 **사이클**이다.\
&rArr; 즉, 함수형 그래프에서 사이클은 거의 구조적으로 반드시 나온다.

---

#### 예시 1
```text
1 -> 2
2 -> 3
3 -> 4
4 -> 2
```
이걸 따라가다 보면
- 1에서 시작
  - 1 - 2 - 3 - 4 - 2\
  즉, `2 - 3 - 4 - 2`가 반복되게 된다.

#### 예시 2
```text
1 → 2
2 → 3
3 → 2

4 → 5
5 → 6
6 → 7
7 → 5

8 → 6
9 → 8
```
구조를 나누어서 보면 다음과 같다:
  - 첫 번째 묶음:
  ```text
  1 → 2 → 3
      ↑   ↓
      └───┘
  ```
  
  - 두 번째 묶음:
  ```text
  4 → 5 → 6 → 7
      ↑       ↓
      └───────┘
  8 → 6
  9 → 8
  ```

## 3) 알고리즘 패턴
### a. 사이클 찾기 (DFS)
- `visited`배열 + `finished`배열 사용
  - `visited` &rArr; 방문 여부
  - `finished` &rArr; DFS 종료 여부
```python
def dfs(x):
    visited[x] = True
    nxt = graph[x]

    if not visited[nxt]:
        dfs(nxt)
    elif not finished[nxt]:
        # 사이클 발견
        ...
    
    finished[x] = True
```

### b. 사이클에 포함된 노드 찾기
- DFS 중에 **되돌아가는 순가**
- 사이클 크기 계산 가능

### c. 사이클까지 거리 구하기
- BFS or DP

## 4) 대표 문제 유형

- 사이클 개수 찾기
- 사이클에 속한 노드 찾기
- 사이클까지 거리 구하기
- 특정 노드가 사이클에 포함되는지
- 방문 경로 추적
