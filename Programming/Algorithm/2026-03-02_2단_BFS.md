# 2단 BFS (불 문제)

---

## 1. 문제 요약

격자에서 **불(F)** 이 먼저 퍼지고,  
사람(J)은 불을 피해 탈출해야 한다.

- 불과 사람은 매 초 동시에 이동한다.
- 사람이 어떤 칸에 도착할 때,
  - 그 칸에 불이 이미 도착했거나
  - 같은 시간에 도착하면 이동할 수 없다.
- 탈출은 격자 밖으로 나가는 순간 성공이다.

### 한 줄 요약

> 시간 기반으로 두 주체가 동시에 움직이는 격자 BFS 문제

---

## 2. 내가 했던 실수들

### (1) 불과 사람을 한 큐에 넣으려 했다

→ 시간 동기화가 꼬이고 조건 처리가 복잡해진다.  
→ 동시에 진행되는 문제처럼 보여도 로직은 분리해야 한다.

---

### (2) 불을 단순 방문처리로만 관리했다

→ 중요한 것은 "불이 왔는지"가 아니라  
→ "불이 언제 도착하는지"이다.

이 문제는 방문 여부가 아니라 **도착 시간 비교 문제**다.

---

### (3) 비교 조건 실수

```python
if person_time + 1 <= fire_time:   # ❌
```
같은 시간에 도착해도 불에 타는 것으로 처리해야 한다.
```python
if person_time + 1 < fire_time:    # ✅
```
이 `<`조건이 문제의 핵심이다.

---

## 3. 왜 꼭 2단 BFS인가?
이 문제는 시간 경쟁 문제다.\

불과 사람은 동시에 움직이지만,\
BFS는 본질적으로 최단 시간 탐색 알고리즘이다.

즉, 동시에 움직이는 것처럼 보이지만\
실제로는 시간을 기준으로 분리해서 계산할 수 있다.

#### 핵심 사고 흐름
1. 불의 도착 시간은 사람과 무관하게 먼저 계산할 수 있다.

2. 불은 최단 거리로 확산된다 → BFS

3. 사람도 최단 거리로 이동한다 → BFS

4. 사람은 이동할 때 “불 도착 시간”과 비교해야 한다.

따라서,
> 먼저 불의 "시간표"를 만든 뒤\
> 그 시간표를 기준으로 사람을 이동시키면 된다.

이것이 2단 BFS의 본질이다.

---

## 4. 2단 BFS 접근 방법

### 1단계: 불 BFS (멀티소스 BFS)

- 시작점이 여러 개 (모든 F)

- `fire[x][y]` = 해당 칸에 불이 도달하는 최소 시간

- 초기값은 `INF`


```python
fire = [[INF] * C for _ in range(R)]

for 모든 F:
    fire[i][j] = 0
    fire_q.append((i, j))
```

→ BFS로 모든 칸의 불 도착 시간을 계산한다.

### 2단계: 사람 BFS

이동 조건:

1. 벽이 아니어야 한다.

2. 아직 방문하지 않은 칸이어야 한다.

3. 다음 시간 `<` 불 도착 시간이어야 한다.

```python
if person_time + 1 < fire[nx][ny]:
```

탈출 조건:

```python
if nx < 0 or nx >= R or ny < 0 or ny >= C:
    return person_time + 1
```
